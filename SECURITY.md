# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка |
|---|---|
| `0.x` (alpha/beta) | Best-effort. Применяем критические патчи в течение недели. |
| `1.x` | Critical-патчи — 1 неделя, Major — 1 месяц. |

## Сообщить об уязвимости

**Не открывайте публичный Issue для уязвимостей.**

Используйте [GitHub Private Vulnerability Reporting](https://github.com/flowa7021-source/Reader/security/advisories/new). Альтернатива — email мейнтейнерам (адрес добавим, когда сформирована команда).

В отчёте опишите:

1. Тип уязвимости (RCE / path traversal / XXE / DoS / data leak / ...).
2. Затрагиваемая версия (или коммит).
3. Шаги для воспроизведения.
4. Минимальный repro (PoC).
5. Возможный impact.

## Что мы делаем

| Шаг | Срок (целевой) |
|---|---|
| Подтверждение получения | 48 часов |
| Первичная триажная оценка | 5 рабочих дней |
| Патч + advisory | По severity (CVSS): Critical — 7 дней, High — 14, Medium — 30, Low — best-effort |
| CVE | Запрашиваем, если применимо |

## Что считаем уязвимостью

- Удалённое выполнение кода через специально сформированный документ (PDF/DjVu/EPUB/...).
- Чтение/запись файлов вне sandbox при обработке документа.
- Утечка содержимого защищённых паролем документов через кэш или временные файлы.
- Обход проверки лицензии или anti-tamper триала **через подделку структур, не через очевидные доступы пользователя к файлам своего профиля**.
- DoS, реалистично достижимый специально сформированным документом ≤ 100 МБ.

## Что **не** считаем уязвимостью

- Пользовательские правки `%APPDATA%\Foliant\` собственного профиля.
- «Обход» offline-валидации лицензии через изменение системного времени (детектится, но не уязвимость).
- Параметры по умолчанию, которые пользователь явно может изменить в Settings.

## Безопасный дефолт

См. раздел 5.7 [`IMPLEMENTATION_PLAN.md`](IMPLEMENTATION_PLAN.md):
- NTFS ACL для `Cache/`, `Autosave/`, `Logs/` — Current User.
- DPAPI для секретов (license, trial).
- Защищённые паролем PDF не кэшируются по умолчанию.
