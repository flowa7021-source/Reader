# Проект: Foliant

> Настраиваемая Windows-альтернатива Adobe Acrobat Pro без AI, с акцентом на OCR, inline-редактирование PDF (включая сканы), точную конвертацию в DOCX/XLSX и полноценную поддержку DjVu (как опциональный плагин).

**Документ:** Табло проекта (живой документ, обновляется по мере принятия решений)
**Платформа:** Windows 10 (21H2+) и Windows 11, x64 only (ARM64 — после 1.5)
**Поставка:** Оффлайн multi-tier инсталлятор (Inno Setup, EXE)
**Стек:** C# / .NET 10 LTS / WPF + MVVM (CommunityToolkit.Mvvm)
**Лицензионная модель:** Open-core (MIT для ядра, proprietary для Pro-функций)
**Бизнес-модель:** Freemium (база бесплатно, Pro по разовой лицензии)

**Статус:** Концепт v0.2 — все 68 + 1 стратегических вопросов закрыты. Готов к Фазе 0 (подготовка).

## 0. Reality check — главное (читать первым)

В ходе опроса по 68 вопросам выбраны преимущественно «Полные» варианты по каждой функции — это дает продукт уровня **Foxit PDF Editor + ABBYY FineReader комбинированно**. Подсчет реального объема работ:

| Подсистема | Чел-мес |
|---|---|
| Базовая инфраструктура (просмотр + кэш + поиск + UI + локализация + темы + инсталлятор + лицензирование) | 15-20 |
| Inline-редактор PDF (полный reflow + font matching + изображения + OCG + сканы) | 13-19 |
| XFA с JavaScript-движком | 6-9 |
| PDF/A + PDF/UA + PDF/X | 6-9 |
| Конвертеры (DOCX/XLSX/EPUB/CSV + обратная) | 6-7 |
| Аннотации (полный набор + FDF/XFDF) | 3.5-4 |
| Цифровые подписи X.509 + PAdES B+T | 3-4.5 |
| Batch + redaction + шифрование AES-256 + permissions | 5-7 |
| DjVu плагин (рендер + OCR + редактирование метаданных и слоя) | 2.5-3.5 |
| Event-sourced undo + open-core MEF архитектура | 5-7 |
| **ИТОГО для полного 1.0** | **~65-90 чел-мес** |

**С AI-агентами (×1.5-×2 ускорение): 40-55 календарных месяцев для соло-разработчика.** Это **~3-4 года** до полного 1.0.

Цель «6 месяцев до альфы» физически достижима только с **резко урезанным scope для альфы** (см. Фазу 1 в разделе 9). Полный заявленный функционал — это горизонт **3-4 лет** при текущих ресурсах (1 разработчик + AI).

**Три реалистичных стратегии:**

1. **A. Принять долгий горизонт.** Альфа за 6 мес = 25-30% scope. 1.0 через 3-4 года. Минимум команды, максимум реализма.
2. **B. Вычеркнуть «дорогие» фичи навсегда или до 2.0.** XFA с JS, PDF/UA, PDF/X, ГОСТ-подписи — наибольшие кандидаты. Это дает **15-20 чел-мес экономии**, делая 1.0 достижимым за 2-2.5 года соло.
3. **C. Найти второго разработчика для критической ветки** — редактор PDF (самая дорогая и долгая часть). Это сокращает срок до 1.0 примерно вдвое.

→ **Это решение лежит за пределами концепта** и должно быть принято до старта Фазы 1.

## 1. Концепт продукта

### 1.1. Видение в одном предложении

**Foliant** — легкое, быстрое, полностью оффлайн-приложение для Windows, которое закрывает 90% реальных задач Adobe Acrobat Pro (просмотр, редактирование, OCR, конвертация, формы, аннотации, безопасность) — плюс полноценная поддержка DjVu в том же интерфейсе и теми же инструментами.

### 1.2. Чем НЕ является (вычеркнуто навсегда)

- **Не AI-продукт** — никаких LLM, генерации, суммаризации (Q-G12)
- **Не облачный сервис** — никакой синхронизации, совместной работы, удаленного хранения (Q-G12)
- **Не имеет форм-дизайнера** — создание форм с нуля выходит за scope (Q-G12)
- **Не браузерное/web-приложение** — только нативный десктоп
- **Не подписка** — разовая лицензия freemium (Q-G7)
- **Не клон UI Acrobat** — функциональный паритет, но интерфейс собственный

### 1.3. Три УТП продукта (Q-G11)

1. **Единственный с полным DjVu OCR + tooling на уровне PDF** — через опциональный плагин (для ученых, архивов, библиотек)
2. **Самый легкий и быстрый** при паритете функций с Acrobat/Foxit — целевые показатели в разделе кэша
3. **Полностью оффлайн, без AI, без подписки** — редкость в 2026 году, важно для прайваси и противников SaaS

### 1.4. Целевая аудитория (Q-G1)

Универсальный продукт для **всех четырех групп** одновременно:
- **Юристы / нотариусы** → формы, цифровые подписи, redaction
- **Ученые / студенты** → OCR, DjVu, аннотации, EPUB-экспорт
- **Бухгалтеры** → таблицы, экспорт в XLSX/CSV
- **Дизайнеры / издатели** → точная верстка, PDF/X (Phase 4)

### 1.5. Целевые рынки и приоритеты (Q-G2)

**РФ как первый рынок, потом глобальный.** Следствия:
- Кириллический OCR — приоритет 1 (рус + eng в базе, СНГ языки в Standard инсталляторе)
- Локализация UI: RU + EN на старте, остальные позже
- ГОСТ-подписи — **отложены** (выбран только X.509, см. раздел 12 «Архитектурный долг»)

## 2. Лицензионная модель — РЕШЕНО

**Q-A1: Чистый MIT/Apache.** Никаких GPL/AGPL компонентов в основном продукте. DjVu и LibreOffice — через **отдельные опциональные плагины**, которые пользователь устанавливает добровольно (тогда GPL не «заражает» основной продукт).

**Q-A2: Open-core.** Ядро Foliant — под MIT, доступен на GitHub. Pro-функции — закрытый код, продаются по freemium-модели.

### 2.2. Защита от пиратства (Q-G9)

**Легкая offline-валидация:**
- ECDSA подпись лицензионного ключа (приватный ключ на стороне разработчика, публичный — в приложении)
- Локальная проверка формата и подписи
- Список блокировки в обновлениях приложения (раз в релиз)
- Без привязки к hardware ID
- Триал: 30-дневный полный доступ к Pro-функциям без ограничений (Q-G8)
- Anti-tamper для триала: дата начала через DPAPI в `%APPDATA%` + дублирование в реестре `HKCU` + хэш «первого запуска»

### 3.3. Размер инсталляторов (multi-tier)

| Tier | Содержит | Размер |
|---|---|---|
| **Базовый** (по умолчанию) | Ядро + .NET 10 + Tesseract LSTM (рус+eng) + WebView2 | ~600 МБ |
| **Standard** | + СНГ языки OCR + Европа базовая (укр, бел, каз, deu, fra, spa, ita) | ~750 МБ |
| **Full** | + CJK + арабский + иврит | ~950 МБ |
| **Pro-плагины** (отдельные .exe, скачиваются с сайта) | DjVu плагин (~5 МБ), LibreOffice плагин (~350 МБ) | — |

Каждый tier — **полностью автономный**, без дозагрузки. Это соответствует требованию «все в инсталляторе без дозагрузки».

### 3.4. Модель плагинов (Q-A8)

**MEF (Managed Extensibility Framework)** для Pro-плагинов:
- In-process DLL в `%ProgramFiles%\Foliant\Plugins\`
- Discovery через `DirectoryCatalog`
- Контракты (interfaces) в открытом ядре, реализации в закрытых Pro-DLL

**Out-of-process плагины** (DjVu, LibreOffice) — **per-call** запуск процесса (Q-A7):
- Простая модель, краш не валит UI
- IPC overhead ~50ms (DjVu) — ~500ms (LibreOffice)
- При необходимости — миграция на гибрид (persistent worker для рендера) в Phase 3

### 4.1. Единая абстракция документа

```csharp
public interface IDocument : IAsyncDisposable
{
    DocumentKind Kind { get; }      // Pdf | Djvu | Image | Epub | Fb2 | Mobi
    int PageCount { get; }
    DocumentMetadata Metadata { get; }
    Task<IPageRender> RenderPageAsync(int pageIndex, RenderOptions opts, CancellationToken ct);
    Task<TextLayer?> GetTextLayerAsync(int pageIndex, CancellationToken ct);
    IDocumentEditor? GetEditor();   // null если read-only (EPUB/FB2/MOBI)
    IFormController? GetForms();    // null если форм нет
    ISignatureController? GetSignatures();
    // ...
}
```

UI и ViewModel **не знают**, читают они PDF, DjVu или EPUB. Любой документ обрабатывается одинаково.

| Дополнительные форматы экспорта | EPUB + JPEG/PNG/TIFF постранично + CSV (для таблиц) | Q-F27 |
| Обратная конвертация в PDF/DjVu | **Полная**: DOCX/XLSX/PPTX → PDF (через LibreOffice плагин) + изображения → PDF/DjVu + HTML → PDF (через WebView2) | Q-F28 |
| Batch-обработка | **Полная**: drag-and-drop сотен файлов + параметры + параллельный run + прогресс + лог | Q-F29 |

### 5.8. Безопасность

| Функция | Решение | Q-* |
|---|---|---|
| Шифрование PDF | **Только современное**: 256-bit AES (PDF 2.0) | Q-F30 |

| Redaction | **Полное**: физическое удаление текста/изображений + черные прямоугольники + find-and-redact | Q-F32 |

## 6. Кэш — детальная архитектура

### 6.1. Зачем кэшировать

| Что повторяется | Стоимость пересчета | Решение |
|---|---|---|
| Рендер одной страницы (zoom, scroll) | 50–500 мс | In-memory page bitmap cache |
| Текстовый слой (для поиска и копирования) | 10–100 мс на страницу | In-memory text cache |
| Превью-thumbnails | 30–200 мс на страницу | Persistent thumb cache |
| OCR страницы скана | 1–10 секунд | **Persistent OCR cache** (главная экономия) |
| Конвертация PDF→DOCX | 5–60 секунд | Persistent conversion cache |
| Полнотекстовый индекс | минуты | Persistent SQLite FTS5 index |
| Декодирование DjVu страницы | 100–500 мс | In-memory page bitmap cache |

### 6.2. Пять слоев кэша

**Слой 1: In-memory hot cache (RAM, LRU)** — bitmap'ы недавно отрисованных страниц при текущем zoom. Размер: `min(15% RAM, 1 ГБ)`. LRU + sticky для текущей и соседних страниц (±5).

**Слой 2: In-memory thumbnail cache** — превью 96×128 px, всегда в памяти пока документ открыт.

**Слой 3: In-memory text/structure cache** — текстовые слои + дерево структуры (outline, bookmarks, links).

**Слой 4: Persistent disk cache** в `%LOCALAPPDATA%\Foliant\Cache\`:
```
Cache/
├── pages/        — отрендеренные страницы (PNG, lossless)
├── thumbs/       — thumbnails в WebP
├── ocr/          — результаты OCR (JSON.gz: текст + bounding boxes + confidence)
├── conversions/  — закэшированные DOCX/XLSX результаты
├── index/        — SQLite FTS5
└── metadata.db   — учетный SQLite
```

**Слой 5: Persistent search index** — SQLite FTS5, общий индекс для всех недавно открытых документов.

### 6.3. Ключи и инвалидация

```
sha256(file_bytes_first_64KB || file_size || lastWriteTime_utc)
  + ":" + pageIndex
  + ":" + render_engine_version
  + ":" + zoom_bucket    (округление до 25%)
  + ":" + flags          (annotations on/off, dark mode и т.п.)
```

Lightweight fingerprint (первые 64 КБ + размер + время) вместо хэша всего файла. Раз в N открытий — фоновое вычисление полного хэша.

**Триггеры инвалидации:**
1. fingerprint файла изменился → выкидываем все записи
2. версия рендер-движка обновилась → постепенный LRU-eviction
3. версия OCR-движка/моделей изменилась → переиндексация в фоне
4. ручная очистка
5. достижение лимита размера → LRU

### 6.4. Лимиты

- **Жесткий потолок диска**: 5 ГБ default, настраиваемо 500 МБ — 50 ГБ
- **Жесткий потолок RAM**: `min(15% системной, 1 ГБ)`, не меньше 128 МБ
- **Per-document cap**: один документ ≤ 30% от общего кэша
- **Memory pressure**: подписка на `QueryMemoryResourceNotification` → освобождение 50% RAM-кэша при сигнале

### 6.5. Прогрев и предсказание

- Pre-render N±1, N-1 при просмотре страницы N
- Prefetch первых 5 + последних 2 страниц при открытии
- Фоновый OCR для сканов с низким приоритетом

### 6.6. UI кэша

- Settings: размер кэша, кнопка «Очистить», слайдер лимита, чекбокс «Не кэшировать этот документ»
- Контекстное меню Recents: «Удалить из кэша»
- Опция «Очищать кэш при выходе» (off по умолчанию)

### 6.7. Безопасность кэша

- NTFS ACL (доступ только текущему пользователю)
- Защищенные паролем PDF: **не кэшируются** по умолчанию (опция: DPAPI-шифрованный кэш)
- При удалении из Recents — спрос «удалить и кэш?»

### 6.8. Cache vs Auto-save

- **Cache** — derivable, удалив, теряем только скорость
- **Auto-save** — НЕ derivable. Незакоммиченные правки. Отдельный каталог `%LOCALAPPDATA%\Foliant\Autosave\`. Привязан к event-sourced модели undo (Q-A11): каждое событие персистится мгновенно.

### 7.4. Стратегия обновлений (Q-S6, Q-S12)

- При запуске: проверка через GitHub Releases API (раз в день максимум)
- Уведомление: «Доступна v X.Y.Z [Подробнее] [Позже]»
- Кнопка ведет на сайт, на страницу скачивания нового EXE
- Установщик при апгрейде делает **auto-backup настроек, лицензии, autosave** в `%LOCALAPPDATA%\Foliant\Backup\v{version}_{timestamp}\`
- Архивная страница на сайте: список всех релизов с changelog и прямыми ссылками
- Откат: пользователь скачивает старую версию из архива, ставит поверх → Inno Setup восстанавливает совместимые настройки из бэкапа

### 7.5. Code signing

**Обязательно:** EV Code Signing certificate (~250-500 тыс ₽/год) или Standard (~50-80 тыс ₽/год).
- Без подписи: SmartScreen блокирует, антивирусы выдают false positives, пользователи массово отказываются устанавливать
- Подписывается основной EXE + плагины + uninstaller
- Бюджет (Q-G4): пока «не определено» — **критический риск**, см. Раздел 11

## 8. Бренд (Q-G3)

**Рабочее название: Foliant**

**Перед инвестированием в дизайн логотипа и code signing CN — обязательные проверки:**

- `%LOCALAPPDATA%\Foliant\`, `%APPDATA%\Foliant\`, `%ProgramFiles%\Foliant\`

## 12. Архитектурный долг (отложено)

### 12.1. ГОСТ-подписи (КриптоПро / VipNet)

**Контекст**: Q-F25 = только X.509. Это противоречит выбранной аудитории (Q-G1 «юристы/нотариусы») в РФ-сегменте.

**План**: Phase 4+ как Pro-плагин `Foliant.Pro.GostSignatures`.

**Сложность**:
- PInvoke к КриптоПро CSP API (CAPI/CAPILite)
- Специфические OID для ГОСТ-2012-256/512 алгоритмов
- Сертификация криптосредства в ФСБ России (для коммерческой продажи)
- ~3-5 чел-мес работы + сертификация (отдельный длительный процесс)

### 12.2. XFA с JavaScript

**Контекст**: Q-F22 = «Полная с JS», но XFA deprecated Adobe с 2017 и удален из PDF 2.0. Низкая ценность для РФ-аудитории (российские формы — AcroForm).

**Рекомендация**: **переоценить решение в Фазе 0**. Если оставляем — Phase 4 как Pro-плагин `Foliant.Pro.Forms.Xfa`. Если вычеркиваем — экономия 6-9 чел-мес.

### 12.3. Mobile / Mac / Linux версии

**Контекст**: Q-G12 — НЕ вычеркнуты явно (только AI, облако, форм-дизайнер). Архитектура с MVVM + сервисами за интерфейсами **позволяет** миграцию на Avalonia в будущем.

**Решение**: не приоритет в первые 24-36 месяцев, но архитектурно возможно. Не закладываем активно, но и не блокируем.

### 12.4. Сетевая совместная работа

**Контекст**: Q-G12 — облако вычеркнуто. Event-sourced undo (Q-A11) **технически готов** к будущему collaborative editing через CRDT/OT, но это вне scope продукта.

**Решение**: оставить заготовку, не использовать.

### 12.5. PDF/UA + PDF/X

**Контекст**: Q-A10 = «Полная», но это Phase 4 (~4-6 чел-мес).

**Возможный пересмотр**: если 1.0 нужно скорее — выпустить с PDF/A only, добавить PDF/UA + PDF/X в 1.5/2.0.
